# Claude SEO: Multi-Platform Agent Instructions

> For **OpenAI Codex CLI**, **Cursor**, **Gemini CLI**, **Cline**, **Aider**,
> and any other agent harness that reads project-root agent instructions.
>
> This repo vendors the [claude-seo](https://github.com/AgriciDaniel/claude-seo)
> plugin (v2.2.4) as project-level skills under `.claude/`. Claude Code
> discovers them automatically; other harnesses should follow this file.

## Layout in this repo

The upstream plugin's directories are vendored here as:

```
.claude/skills/               # 25 sub-skills (SKILL.md each)
  seo/SKILL.md                # Main orchestrator + command routing
  seo/bin/claude-seo          # Runtime launcher (bash; use for ALL bundled scripts)
  seo/scripts/                # 53 Python scripts, including the managed runtime
  seo/schema/                 # JSON-LD templates
  seo/references/             # On-demand reference docs
  seo/extensions/             # Extension references/scripts (DataForSEO, Banana, ...)
  seo-audit/ seo-page/ seo-technical/ seo-content/ seo-schema/
  seo-sitemap/ seo-images/ seo-geo/ seo-local/ seo-maps/ seo-plan/
  seo-hreflang/ seo-google/ seo-backlinks/ seo-cluster/ seo-sxo/
  seo-drift/ seo-ecommerce/ seo-programmatic/ seo-competitor-pages/
  seo-content-brief/ seo-flow/ seo-dataforseo/ seo-image-gen/ ...
.claude/agents/               # 18 subagent definitions
```

## Runtime

One-time per machine (requires Python 3.10+; bash shell such as Git Bash on
Windows):

```bash
.claude/skills/seo/bin/claude-seo setup    # creates isolated venv + Chromium
.claude/skills/seo/bin/claude-seo doctor   # verify: expect "Runtime: ready"
```

Run all bundled Python tools through the launcher — never a bare interpreter:

```bash
# Page fetching with SSRF protection
.claude/skills/seo/bin/claude-seo run fetch_page.py https://example.com

# HTML parsing for SEO elements
.claude/skills/seo/bin/claude-seo run parse_html.py https://example.com

# PageSpeed Insights
.claude/skills/seo/bin/claude-seo run pagespeed_check.py https://example.com --json
```

Runtime artifacts (`.claude/skills/seo/.venv/`, `ms-playwright/`,
`runtime-state.json`) are gitignored and regenerated by `setup`.

## Quick Reference

| Command | What it does |
|---------|-------------|
| `/seo audit <url>` | Full website audit with parallel subagent delegation |
| `/seo page <url>` | Deep single-page analysis |
| `/seo technical <url>` | Technical SEO audit (9 categories) |
| `/seo content <url>` | E-E-A-T and content quality analysis |
| `/seo content-brief <topic>` | Generate a content brief for a topic |
| `/seo schema <url>` | Schema.org detection, validation, generation |
| `/seo sitemap <url>` | XML sitemap analysis or generation |
| `/seo images <url>` | Image SEO: on-page audit, SERP analysis, file optimization |
| `/seo geo <url>` | AI Overviews / Generative Engine Optimization |
| `/seo plan <type>` | Strategic SEO planning |
| `/seo cluster <keyword>` | SERP-based semantic clustering and content architecture |
| `/seo sxo <url>` | Search Experience Optimization: page-type analysis, personas |
| `/seo drift baseline <url>` | Capture SEO baseline for change monitoring |
| `/seo drift compare <url>` | Compare current state to stored baseline |
| `/seo ecommerce <url>` | E-commerce SEO: product schema, marketplace intelligence |
| `/seo programmatic [url]` | Programmatic SEO at scale |
| `/seo competitor-pages [url]` | Competitor comparison pages |
| `/seo flow [stage]` | FLOW framework prompts (Find, Leverage, Optimize, Win, Local) |
| `/seo local <url>` | Local SEO analysis (GBP, citations, reviews) |
| `/seo maps [cmd] [args]` | Maps intelligence (geo-grid, GBP audit, competitors) |
| `/seo hreflang <url>` | Hreflang/i18n SEO audit, cultural profiles, content parity |
| `/seo google [cmd] [url]` | Google SEO APIs (GSC, PageSpeed, CrUX, Indexing, GA4) |
| `/seo backlinks <url>` | Backlink profile analysis |
| `/seo setup` | Create or refresh the isolated Python runtime and Chromium |
| `/seo doctor` | Check runtime readiness without changing the system |

Harnesses without slash commands: treat `/seo <cmd> <arg>` as a text prompt.
Read `.claude/skills/seo/SKILL.md` for routing, then load the matching
`.claude/skills/seo-<cmd>/SKILL.md` and follow its instructions.

## Key Principles

1. **Progressive Disclosure**: Read SKILL.md for routing, load references on demand
2. **Industry Detection**: Auto-detect SaaS, e-commerce, local, publisher, agency
3. **Security**: All scripts call `validate_url()` for SSRF protection
4. **Config location**: `~/.config/claude-seo/` for API credentials
